KEY FEATURES EXPLAINED
================================================================================

AUTOMATIC TOKEN REFRESH
When access token expires, the system:
1. Detects 401 response from API
2. Calls POST /auth/refresh to get new token
3. Retries original request with new token
4. User experiences no interruption

CONCURRENT REQUEST HANDLING
If multiple requests fail with 401 simultaneously:
1. All requests are queued
2. Token is refreshed only once (not multiple times)
3. All queued requests are retried with new token
4. Prevents refresh flooding, improves efficiency

RATE LIMITING
If server returns 429 (too many requests):
1. First retry: wait 1 second
2. Second retry: wait 2 seconds
3. Third retry: wait 4 seconds
4. Gives server time to recover

PROTECTED ROUTES
Routes in (protected) group automatically:
1. Check if user is authenticated
2. Show loading spinner if checking
3. Redirect to /login if not authenticated
4. Render page if authenticated
